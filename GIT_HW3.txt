Чтобы пригласить кого-то в свой проект и дать доступ к репозиторию, можно выполнить следующие шаги:

1. Откройте страницу вашего репозитория на GitHub.
2. Нажмите на вкладку "Settings" (Настройки) в верхней части страницы.
3. В боковом меню выберите "Manage access" (Управление доступом).
4. Нажмите на кнопку "Invite a collaborator" (Пригласить сотрудника).
5. Введите имя пользователя или адрес электронной почты коллеги, которого вы хотите пригласить, и нажмите на кнопку "Add collaborator" (Добавить сотрудника).
6. GitHub отправит приглашение коллеге, и они получат уведомление о приглашении в ваш репозиторий.

После того, как коллега примет приглашение и присоединится к вашему проекту, они смогут работать с репозиторием и создавать ветки, за исключением ветки master.

Чтобы поставить коллеге задачу и попросить выполнить ее в отдельной ветке, вы можете использовать функцию Issues на GitHub:

1. Откройте страницу вашего репозитория на GitHub.
2. Нажмите на вкладку "Issues" (Задачи) в верхней части страницы.
3. Нажмите на кнопку "New issue" (Новая задача).
4. Введите заголовок и описание задачи. Укажите, что коллега должен выполнить задачу в отдельной ветке.
5. Нажмите на кнопку "Assignees" (Исполнители) и выберите коллегу, которому вы хотите назначить эту задачу.
6. Нажмите на кнопку "Labels" (Метки) и выберите подходящую метку для вашей задачи, если необходимо.
7. Нажмите на кнопку "Submit new issue" (Создать новую задачу) для создания задачи и уведомления коллеги.

Когда коллега выполнил задачу, они могут создать pull request (запрос на внесение изменений) для объединения своей ветки с вашей основной веткой. Вы сможете проверить изменения и принять pull request следующим образом:

1. Откройте страницу вашего репозитория на GitHub.
2. Перейдите на вкладку "Pull requests" (Запросы на внесение изменений) в верхней части страницы.
3. Найдите соответствующий pull request, который создал коллега, и откройте его.
4. Просмотрите изменения, сделанные коллегой, и убедитесь, что они соответствуют ожидаемым результатам.
5. Если изменения корректны, нажмите на кнопку "Merge pull request" (Объединить запрос на внесение изменений) для внесения изменений в вашу основную ветку.
6. На следующем экране нажмите на кнопку "Confirm merge" (Подтвердить объединение), чтобы окончательно принять изменения.
7. После объединения pull request вы можете удалить ветку, которая использовалась для решения задачи.
